￼# fz-tmn

База дампов ключей домофонов, шлагбаумов и других систем контроля доступа в городе Тюмени для использования с Flipper Zero и совместимыми устройствами.

В репозитории хранятся файлы подъездов и точек доступа с расширениями:

- NFC: `.nfc`
- RFID / брелки: `.rfid`
- iButton / OneWire: `.ibutton`
- Sub-GHz: `.sub`

> Все данные предназначены только для законного использования: резервное копирование собственных ключей, тестирование систем, исследовательские задачи. Использование для несанкционированного доступа запрещено.

## 1. Структура репозитория

Базовый уровень — улица и дом:

```text
/<ulica>/<dom>/

где:
	•	<ulica> — транслитерация названия улицы латиницей, слова разделяются _, всё в нижнем регистре;
	•	<dom> — номер дома без дополнительных символов.

Примеры:

50_let_vlksm/57/
anatoliya_klinduka/6/
vancetti/5/
vasiliya_goltsova/22/

Внутри папки дома хранятся:
	•	дампы подъездов — прямо в корне папки дома;
	•	при необходимости — подпапки для шлагбаумов и прочего:
	•	gate/ — шлагбаумы, шлагбаумы на въезды, парковки и т.д.;
	•	service/ — служебные входы, техпомещения и прочее нетипичное.

Норма:

/<ulica>/<dom>/
  1.rfid
  2.nfc
  3.sub
  gate/
    shlagbaum_v_zdanie.sub
    shlagbaum_na_parking.sub
  service/
    cherdak_dver.ibutton
    tech_pomeshchenie.rfid

2. Правила именования файлов

2.1. Подъезды (основной кейс)

Имя файла подъезда — это только номер подъезда:

<porch>.<ext>

где:
	•	<porch> — номер подъезда: 1, 2, 3 и т.д.;
	•	<ext> — одно из: nfc, rfid, ibutton, sub.

Примеры:

1.rfid      # 1-й подъезд
2.nfc       # 2-й подъезд
3.sub       # 3-й подъезд, Sub-GHz
4.ibutton   # 4-й подъезд, iButton

Если для одного подъезда есть несколько разных дампов (разные ключи / разные конфиги), используется суффикс:

1_1.rfid
1_2.rfid
2_1.sub

Базовая часть имени всегда начинается с номера подъезда.

2.2. Нетипичные точки доступа (gate, service и т.п.)

Нетипичные точки (шлагбаумы, служебные двери, парковки, въезды) складываются в подпапки gate/ и service/ дома.

Имена файлов — краткое описание латиницей, через _, максимально понятно, без лишней воды:

Примеры для gate/:

gate/
  shlagbaum_v_zdanie.sub
  shlagbaum_na_parking.sub
  vyezd_iz_dvora.sub

Примеры для service/:

service/
  sluzhebnaya_dver.rfid
  tech_pomeshchenie_1.ibutton
  cherdak_dostup.rfid

Нормы:
	•	только латиница;
	•	слова разделяются _;
	•	без пробелов и кириллицы;
	•	по имени понятно, что это и где примерно находится.

⸻

3. Типы файлов и устройства

В репозитории могут лежать дампы следующих типов:
	•	.nfc — дампы NFC-ключей;
	•	.rfid — дампы RFID-брелков и карт;
	•	.ibutton — дампы iButton (OneWire);
	•	.sub — дампы Sub-GHz сигналов (например, шлагбаумы, радиобрелки).

Главное требование — файл должен быть корректно читаем устройством (Flipper Zero или аналог) и реально открывать/управлять целевой точкой доступа.

4. Добавление новых данных

Стандартный процесс:
	1.	Определить адрес:
	•	улица → транслитерация в формате <ulica>;
	•	дом → папка <dom>.
	2.	Перейти в папку дома (создать, если её нет):

mkdir -p <ulica>/<dom>
cd <ulica>/<dom>


	3.	Снять дамп(ы) с Flipper Zero / другого устройства.
	4.	Переименовать файлы под принятые правила:
	•	для подъезда — 1.rfid, 2.nfc, 3.sub и т.п.;
	•	для шлагбаумов и прочего:
	•	gate/shlagbaum_v_zdanie.sub;
	•	gate/shlagbaum_na_parking.sub;
	•	service/sluzhebnaya_dver.rfid и т.д.
	5.	Добавить файлы в git:

git add .
git commit -m "Добавлены дампы по адресу <ulica> <dom>"
git push



README внутри конкретного дома — опционален. Если он есть, в нём кратко описывается, что именно лежит в папке (например, описание нестандартных точек доступа).

⸻

5. Требования к репозиторию и git

Рекомендуется:
	•	не коммитить служебные файлы ОС:
	•	.DS_Store
	•	Thumbs.db
	•	кэш IDE и т.п.;
	•	следить, чтобы в репозиторий не попадали:
	•	персональные данные жильцов;
	•	телефоны и прочее, что может деанонимизировать людей.

Если появляются новые типы файлов или устройств, их формат и нейминг должны быть описаны здесь же или в отдельной документации, но по тем же принципам: простые имена, минимум сущностей, максимум понятности.

6. Правила использования
	•	Нельзя:
	•	использовать дампы для проникновения в чужие подъезды, дворы, парковки, техпомещения без явного разрешения;
	•	обходить системы безопасности, нарушать закон и частную собственность.
	•	Можно:
	•	хранить резервные копии собственных ключей;
	•	проверять и тестировать системы доступа там, где у вас есть права;
	•	использовать в рамках исследований, разработки и аудита инфраструктуры.

Ответственность за применение данных из репозитория всегда на пользователе.
