# fz-tmn

База дампов ключей домофонов, шлагбаумов и других систем контроля доступа в городе Тюмени для использования с Flipper Zero и совместимыми устройствами.

База создаётся для:
- хранения дампов подъездов, шлагбаумов, служебных точек доступа;
- систематизации данных по адресам;
- упрощения поиска нужного дома, подъезда или устройства;
- исследования инфраструктуры доступа.

Поддерживаемые расширения файлов:
- .nfc — NFC
- .rfid — RFID брелки/карты
- .ibutton — iButton / OneWire
- .sub — Sub-GHz сигналы (радиобрелки, шлагбаумы)

Все данные предназначены только для законного использования. Несанкционированный доступ запрещён.

---

## 1. Быстрый старт

1. Найдите улицу (транслитерация латиницей, через `_`).
2. Откройте папку дома.
3. Дампы подъездов лежат прямо в корне папки.
4. Шлагбаумы — всегда внутри `gate/`.
5. Служебные точки — внутри `service/`.
6. Файлы подъездов называются просто номером: `1.rfid`, `2.sub` и т.д.

---

## 2. Структура репозитория

Каждый адрес представлен директорией:

<ulica>/<dom>/

Где:
- <ulica> — транслитерация улицы, нижний регистр, слова через `_`;
- <dom> — номер дома.

Пример:

50_let_vlksm/57  
anatoliya_klinduka/6  
vancetti/5  
vasiliya_goltsova/22

---

## 3. Пример структуры дома (визуально)

ulica/dom/  
├── 1.rfid  
├── 2.nfc  
├── 3.sub  
├── gate/  
│   ├── shlagbaum_v_zdanie.sub  
│   └── shlagbaum_na_parking.sub  
└── service/  
    ├── sluzhebnaya_dver.ibutton  
    └── tech_pom_1.rfid

---

## 4. Правила транслитерации улиц

- только латиница;  
- пробелы → `_`;  
- `ё` → `e`;  
- `й` → `y`;  
- схемы пунктуации удаляются;  
- всё в lower-case.

Примеры:
- ул. Вацетти → vancetti  
- ул. Анатолия Клиндюка → anatoliya_klinduka  

---

## 5. Именование файлов

### Подъезды
Файл называется **только номером подъезда**:

<porch>.<ext>

Примеры правильных имён:
1.rfid  
2.nfc  
3.sub  
4.ibutton

Если несколько дампов:
1_1.rfid  
1_2.rfid  
2_1.sub

---

### Gate / Service
Нестандартные точки доступа — только в подпапках.

gate/  
• shlagbaum_v_zdanie.sub  
• shlagbaum_na_parking.sub  
• vyezd_iz_dvora.sub  

service/  
• sluzhebnaya_dver.rfid  
• tech_pom_1.ibutton  
• cherdak_dostup.rfid  

Правила:
- кратко и понятно;
- только латиница;
- без пробелов, только `_`.

---

## 6. Хорошие и плохие примеры имён файлов

| Тип                | Правильно                         | Неправильно                                 |
|-------------------|------------------------------------|----------------------------------------------|
| Подъезд           | `1.rfid`                           | `подъезд1.nfc`                               |
| Подъезд (несколько)| `1_2.sub`                         | `1-final(sub).sub`                           |
| Gate              | `shlagbaum_v_zdanie.sub`           | `шлагбаум в здание.sub`                      |
| Gate              | `vyezd_iz_dvora.sub`               | `gate_file_final_v3.sub`                     |
| Service           | `sluzhebnaya_dver.rfid`            | `служебка.rfid`                              |
| Service           | `tech_pom_1.ibutton`               | `техпомещение_1.ibutton`                     |
| Подъезд           | `3.nfc`                            | `3й_подъезд.nfc`                             |
| Любой тип         | `2.sub`                            | `2 (копия) — исправленный.sub`               |

Таблица помогает унифицировать стиль и исключить мусорные имена.

## 7. Добавление новых данных (чеклист)

1. Создать папку адреса, если отсутствует:  
   <ulica>/<dom>/

2. Поместить файлы подъездов в корень папки.

3. Поместить шлагбаумы в директорию:  
   <ulica>/<dom>/gate/

4. Поместить служебные точки:  
   <ulica>/<dom>/service/

5. Проверить имена файлов по правилам.

6. Закоммитить изменения:  
   git add .  
   git commit -m "Добавлены дампы по адресу <ulica> <dom>"  
   git push

---

## 8. Требования к чистоте репозитория

- Не добавлять `.DS_Store`, `Thumbs.db` и мусор.  
- Не использовать кириллицу.  
- Не хранить персональные данные жильцов.  
- Использовать только структурированные директории.

---

## 9. Правила использования

Запрещено:
- несанкционированное открытие дверей и шлагбаумов;
- использование данных в противоправных целях.

Разрешено:
- резервное копирование собственных ключей;
- исследовательские задачи;
